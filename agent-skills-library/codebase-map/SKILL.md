# Codebase Map

> *This file describes the actual structure of the Flight Tracker project.
> When the structure changes, this file is updated in the same commit.*

---

## Project Identity

- **Name:** `flight-tracker`
- **Purpose (one sentence):** Track flight prices across user-defined routes and dates, log price history, and alert on significant drops.
- **Primary stack:** TypeScript, Next.js (App Router), Vanilla CSS, SQLite
- **Repository layout:** Single application (not a monorepo)

## Where Things Live

| Kind of thing | Where it lives | Naming convention |
|---|---|---|
| Domain entities and types | `src/types/` | PascalCase interfaces: `Route`, `PricePoint` |
| Pure business logic | `src/lib/` | camelCase modules: `tracker.ts`, `priceUtils.ts` |
| Data access / queries | `src/lib/db/` | camelCase: `routes.ts`, `priceHistory.ts` |
| Flight data fetchers | `src/lib/scrapers/` | One file per provider: `serpapi.ts`, `amadeus.ts` |
| Notification adapters | `src/lib/notifications/` | One file per channel: `telegram.ts`, `email.ts` |
| API route handlers | `src/app/api/` | Directory per resource: `api/flights/route.ts` |
| Background jobs / tracker | `src/lib/tracker.ts` | Single entry point for cron logic |
| Database schema | `prisma/schema.prisma` | Prisma schema file |
| Database migrations | `prisma/migrations/` | Auto-generated by Prisma |
| Configuration | `.env.local` + `next.config.js` | Per environment |
| Shared utilities | `src/lib/utils.ts` | Pure functions only |
| Frontend components | `src/components/` | PascalCase `.tsx`: `FlightCard.tsx` |
| Frontend pages / routes | `src/app/` | Next.js App Router convention: `page.tsx` per dir |
| Frontend state / hooks | `src/lib/hooks/` | `use` prefix: `usePriceHistory.ts` |
| Global styles & tokens | `src/styles/` | kebab-case `.css`: `globals.css`, `tokens.css` |
| Unit tests | Co-located or `__tests__/` | `*.test.ts` / `*.test.tsx` |
| Integration tests | `tests/integration/` | `*.test.ts` |
| End-to-end tests | `tests/e2e/` | `*.spec.ts` (Playwright) |

## Naming Conventions

| Thing | Convention | Example |
|---|---|---|
| Component file | PascalCase `.tsx` | `FlightCard.tsx` |
| Library module | camelCase `.ts` | `tracker.ts` |
| Type / Interface | PascalCase | `Route`, `PricePoint`, `AlertConfig` |
| Constant | UPPER_SNAKE_CASE | `MAX_RETRY_COUNT` |
| CSS file | kebab-case `.css` | `flight-card.css` |
| Test file | `*.test.ts` / `*.test.tsx` | `tracker.test.ts` |
| Environment variable | UPPER_SNAKE_CASE | `FLIGHT_API_KEY` |

## Dependency Direction

```
src/app/  (pages, layouts, API routes)
    ↓
src/components/  (presentational, props-driven, no side effects)
    ↓
src/lib/  (business logic, DB, HTTP, notifications)
    ↓
src/types/  (pure type definitions, zero runtime code)
```

Reject changes that violate this direction. See the `architecture`
and `flight-tracker-stack` skills.

## Build, Run, Test

```bash
# Install
pnpm install

# Run dev
pnpm dev

# Tests
pnpm test                         # all tests
pnpm test -- --filter tracker     # single file/pattern

# Lint and format
pnpm lint
pnpm format

# Type-check
pnpm tsc --noEmit

# Database
pnpm prisma migrate dev           # apply migrations
pnpm prisma generate              # regenerate client
pnpm prisma studio                # visual DB browser

# Build for production
pnpm build
pnpm start
```

## Environment

- Required environment variables are listed in `.env.example`.
- Secrets are **never** committed. Use `.env.local` for local dev.
- No external infrastructure required for local dev — SQLite is file-based.
